## FRONTEND
################
# [WRFE-](feat): 새로운 기능 추가
# [WRFE-](fix): 버그 수정
# [WRFE-](docs): 문서 수정
# [WRFE-](test): 테스트 코드, 리팩토링 테스트 코드 추가
# [WRFE-](refactor): 코드 리팩토링
# [WRFE-](style): 코드 formatting, 세미콜론 누락, 코드 자체의 변경이 없는 경우
# [WRFE-](chore): 패키지 매니저 수정, 그 외 기타 수정 ex) .gitignore
# [WRFE-](design): CSS 등 사용자 UI 디자인 변경
# [WRFE-](comment): 필요한 주석 추가 및 변경
# [WRFE-](rename): 파일 또는 폴더 명을 수정하거나 옮기는 작업만인 경우
# [WRFE-](remove): 파일을 삭제하는 작업만 수행한 경우
# [WRFE-](!BREAKING CHANGE): 커다란 API 변경의 이유
# [WRFE-](!HOTFIX): 급하게 치명적인 버그를 고쳐야 하는 경우

################
